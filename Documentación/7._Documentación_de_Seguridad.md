## 2. Autenticación y Autorización

### 2.1 Sistema de Autenticación

#### Contraseñas
| Aspecto | Implementación |
|---------|----------------|
| **Algoritmo** | bcrypt con salt rounds = 12 |
| **Hash** | bcryptjs en Node.js |
| **Almacenamiento** | Solo hash, nunca texto plano |
| **Validación** | Mínimo 6 caracteres |

```typescript
// src/lib/auth.ts
import bcrypt from 'bcryptjs';

export async function hashPassword(password: string): Promise<string> {
  return bcrypt.hash(password, 12);
}

export async function comparePassword(password: string, hash: string): Promise<boolean> {
  return bcrypt.compare(password, hash);
}
```

#### Tokens JWT
| Aspecto | Implementación |
|---------|----------------|
| **Algoritmo** | HS256 (HMAC SHA256) |
| **Access Token** | 7 días de expiración |
| **Refresh Token** | UUID, 7 días, stored in DB |
| **Payload** | {id, email, rol} |

```typescript
// src/lib/auth.ts
import jwt from 'jsonwebtoken';

export function generateAccessToken(user: User): string {
  return jwt.sign(
    { id: user.id, email: user.email, rol: user.rol },
    process.env.JWT_SECRET!,
    { expiresIn: process.env.JWT_EXPIRES_IN || '7d' }
  );
}
```

### 2.2 Sistema de Roles

| Rol | Descripción | Permisos |
|-----|-------------|----------|
| **admin** | Administrador del sistema | Dashboard, productos, categorías, órdenes, configuración, contactos |
| **cliente** | Cliente del restaurante | Menú, canasta, pedidos propios, perfil |
| **repartidor** | Personal de entrega | Actualizar estado de órdenes |

### 2.3 Middleware de Protección

```typescript
// src/middleware/auth.ts
export function withAuth(handler) {
  return async (req: NextRequest) => {
    const token = req.headers.get('authorization')?.split(' ')[1];
    if (!token) {
      return NextResponse.json({ error: 'No autorizado' }, { status: 401 });
    }
    
    try {
      const decoded = verifyAccessToken(token);
      req.user = decoded;
      return handler(req);
    } catch (error) {
      return NextResponse.json({ error: 'Token inválido' }, { status: 401 });
    }
  };
}

export function withRole(role: string) {
  return (handler) => async (req: NextRequest) => {
    if (req.user.rol !== role) {
      return NextResponse.json({ error: 'Acceso denegado' }, { status: 403 });
    }
    return handler(req);
  };
}

export const withAdmin = withRole('admin');
```

## 3. Protección de Datos

### 3.1 Datos Encriptados

| Dato | Protección | Ubicación |
|------|------------|-----------|
| Contraseñas | bcrypt hash | Tabla `usuarios` |

- ❌ Contraseñas de bancos
- ❌ Tokens de bancos

| Dato | Protección |
|------|------------|

Para producción, implementar rate limiting:

```typescript
// Ejemplo con Upstash Rate Limit (para Vercel)
import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '10 s'),
});

export async function rateLimitMiddleware(req: Request) {
  const ip = req.headers.get('x-forwarded-for') || 'anonymous';
  const { success, remaining } = await ratelimit.limit(ip);
  
  if (!success) {
    return new Response('Too Many Requests', { status: 429 });
  }
}
```

## 5. Seguridad de la Base de Datos

### 5.1 Principios de Protección

- ✅ Principio de mínimo privilegio
- ✅ Índices para optimizar consultas
- ✅ Prepared statements (evita SQL injection)
- ✅ Transacciones para operaciones críticas

### 5.2 Conexión a la Base de Datos

```typescript
// src/lib/db.ts
import mysql from 'mysql2/promise';

const pool = mysql.createPool({
  host: process.env.DB_HOST,
  port: parseInt(process.env.DB_PORT || '3306'),
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0,
  // SSL para producción
  ssl: process.env.NODE_ENV === 'production' ? {
    rejectUnauthorized: true
  } : undefined
});
```

### 5.3 Prepared Statements

```typescript
// ✅ CORRECTO - Usa placeholders
const [rows] = await pool.execute(
  'SELECT * FROM usuarios WHERE email = ?',
  [email]
);

// ❌ INCORRECTO - SQL Injection vulnerable
const [rows] = await pool.execute(
  `SELECT * FROM usuarios WHERE email = '${email}'`
);
```

| **Private Key** | Backend (webhooks) | NUNCA exponer |

### 6.3 Validación de Webhook

```typescript
// src/app/api/pagos/webhook/route.ts
import crypto from 'crypto';

export async function verifyWompiSignature(
  payload: string,
  signature: string,
  secret: string
): Promise<boolean> {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(payload)
    .digest('hex');
  
  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}
```

## 9. Procedimientos de Respuesta a Incidentes

### 9.1 Tipos de Incidentes

| Nivel | Tipo | Tiempo de Respuesta |
|-------|------|---------------------|
| **Crítico** | Breach de datos, hackeo | Inmediato |
| **Alto** | Acceso no autorizado, malware | < 1 hora |
| **Medio** | Phishing attempt, bug de seguridad | < 4 horas |
| **Bajo** | Violación de política menor | < 24 horas |

### 9.2 Pasos de Respuesta

1. **Identificar**: Detectar el incidente
2. **Contener**: Limitar el daño
3. **Erradicar**: Eliminar la amenaza
4. **Recuperar**: Restaurar servicios
5. **Documentar**: Registrar lecciones aprendidas

## 10. Auditoría

### 10.1 Logs de Auditoría

El sistema mantiene logs de:

| Evento | Datos Registrados |
|--------|-------------------|
| Login exitoso | Usuario, IP, timestamp |
| Login fallido | Email尝试, IP, timestamp |
| Cambios en órdenes | Admin, orden, cambio, timestamp |
| Cambios en productos | Admin, producto, cambio |
| Transacciones | Orden, monto, método, estado |

### 10.2 Tabla de Auditoría

```sql
CREATE TABLE logs_sistema (
  id_log BIGINT PRIMARY KEY AUTO_INCREMENT,
  tabla VARCHAR(50),
  accion VARCHAR(20),
  registro_id INT,
  datos_anteriores JSON,
  datos_nuevos JSON,
  usuario_id INT,
  ip_address VARCHAR(45),
  user_agent TEXT,
  fecha TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 10.3 Triggers de Auditoría

```sql
-- Auditoría automática de órdenes
CREATE TRIGGER tr_ordenes_audit
AFTER UPDATE ON ordenes
FOR EACH ROW
BEGIN
  INSERT INTO logs_sistema (tabla, accion, registro_id, datos_anteriores, datos_nuevos)
  VALUES ('ordenes', 'UPDATE', NEW.id_orden, JSON_OBJECT('estado', OLD.estado), JSON_OBJECT('estado', NEW.estado));
END;
```
