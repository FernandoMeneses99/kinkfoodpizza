# Plan de Mantenimiento y Soporte - Krokori

## 1. Introducción

Este documento establece el plan de mantenimiento y soporte técnico para el sistema Krokori después de su entrega formal.

### 1.1 Objetivo

Asegurar el funcionamiento continuo, seguro y optimizado del sistema Krokori mediante mantenimiento preventivo, correctivo y evolutivo.

### 1.2 Alcance

- Mantenimiento del código fuente
- Actualización de dependencias
- Respaldo y recuperación de base de datos
- Monitoreo del sistema
- Soporte técnico a usuarios

---

## 2. Tipos de Mantenimiento

### 2.1 Mantenimiento Correctivo

Reparación de defectos o errores encontrados en el sistema.

| Prioridad | Descripción | Tiempo de Respuesta |
|-----------|-------------|---------------------|
| **Crítica** | Sistema caído, pérdida de datos | Inmediato (< 1 hora) |
| **Alta** | Funcionalidad principal afectada | < 4 horas |
| **Media** | Funcionalidad secundaria afectada | < 24 horas |
| **Baja** | Errores menores, mejoras menores | < 72 horas |

### 2.2 Mantenimiento Preventivo

Actividades para prevenir problemas futuros.

| Actividad | Frecuencia | Descripción |
|-----------|------------|-------------|
| Backup de BD | Diaria | Respaldo completo de la base de datos |
| Actualización dependencias | Mensual | Actualizar paquetes de npm |
| Revisión de logs | Semanal | Analizar logs de errores |
| Revisión de rendimiento | Mensual | Optimizar consultas lentas |
| Revisión de seguridad | Trimestral | Auditar vulnerabilidades |

### 2.3 Mantenimiento Evolutivo

Nuevas funcionalidades o mejoras al sistema.

| Tipo | Descripción | Costo |
|------|-------------|-------|
| **Menor** | Ajustes de UI, cambios menores | $[tarifa_hora] / hora |
| **Mayor** | Nuevas funcionalidades | Cotización previa |
| **Custom** | Integraciones especiales | Cotización previa |

## 6. Respaldo y Recuperación

### 6.1 Política de Backups

| Tipo | Frecuencia | Retención | Ubicación |
|------|------------|-----------|-----------|
| **Full** | Diaria | 7 días | Local + Nube |
| **Semanal** | Domingos | 4 semanas | Nube |
| **Mensual** | Primer día | 12 meses | Nube |

### 6.2 Procedimiento de Restauración

1. **Detectar incidente**
2. **Evaluar impacto**
3. **Activar plan de emergencia**
4. **Restaurar desde backup**
5. **Verificar funcionamiento**
6. **Documentar incidente**

### 6.3 RPO y RTO

| Métrica | Valor | Definición |
|---------|-------|------------|
| **RPO** | 4 horas | Punto máximo de pérdida de datos |
| **RTO** | 2 horas | Tiempo máximo de recuperación |
