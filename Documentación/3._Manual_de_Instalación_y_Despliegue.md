## 2. Pasos de Instalación

### 2.1 Clonar el Repositorio

```bash
# Clonar el repositorio
git clone https://github.com/tu-usuario/kinkfoodpizza.git

# Entrar al directorio
cd kinkfoodpizza

# Instalar dependencias
npm install
```

### 2.2 Instalar XAMPP (Windows)

1. Descargar XAMPP desde [apachefriends.org](https://www.apachefriends.org/)
2. Ejecutar el instalador
3. Seleccionar solo **MySQL** y **PHP** (para phpMyAdmin)
4. Instalar en `C:\xampp`
5. Iniciar los servicios **Apache** y **MySQL** desde XAMPP Control Panel

### 2.3 Crear la Base de Datos

#### Opción A: Usando phpMyAdmin

1. Abrir navegador e ir a `http://localhost/phpmyadmin`
2. Click en **"Nueva"** en el panel izquierdo
3. Nombre: `krokori_db`
4. Cotejamiento: `utf8mb4_unicode_ci`
5. Click en **"Crear"**

#### Opción B: Usando MySQL CLI

```bash
mysql -u root -p

# En la consola MySQL
CREATE DATABASE krokori_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE krokori_db;

# Importar el schema
SOURCE C:/Proyectos/kinkfoodpizza/database/krokori_database.sql;

# Salir
EXIT;
```

### 2.4 Configurar Variables de Entorno

```bash
# Copiar el archivo de ejemplo
copy .env.example .env.local
```

Editar `.env.local` con los valores correctos:

```env
# Base de datos (XAMPP)
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=krokori_db

# JWT
JWT_SECRET=krokori-super-secret-key-2024-cambiar-en-produccion
JWT_EXPIRES_IN=7d

# Wompi (Sandbox para pruebas)
NEXT_PUBLIC_WOMPI_PUBLIC_KEY=pub_test_xxxxx
WOMPI_PRIVATE_KEY=prv_test_xxxxx
WOMPI_CURRENCY=COP
WOMPI_ENVIRONMENT=sandbox

# App
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_APP_NAME=Krokori
```

### 2.5 Ejecutar el Servidor de Desarrollo

```bash
# En la raíz del proyecto
npm run dev
```

El servidor estará disponible en: **http://localhost:3000**

## 4. Variables de Entorno Detalladas

### 4.1 Referencia Completa

| Variable | Descripción | Ejemplo |
|----------|-------------|---------|
| `DB_HOST` | Host del servidor MySQL | localhost |
| `DB_PORT` | Puerto MySQL | 3306 |
| `DB_USER` | Usuario MySQL | root |
| `DB_PASSWORD` | Contraseña MySQL | (vacío para XAMPP) |
| `DB_NAME` | Nombre de la base de datos | krokori_db |
| `JWT_SECRET` | Clave secreta para JWT | cadena-larga-aleatoria |
| `JWT_EXPIRES_IN` | Tiempo de expiración del token | 7d |
| `NEXT_PUBLIC_WOMPI_PUBLIC_KEY` | Llave pública Wompi | pub_test_xxx |
| `WOMPI_PRIVATE_KEY` | Llave privada Wompi | prv_test_xxx |
| `WOMPI_CURRENCY` | Moneda de transacciones | COP |
| `WOMPI_ENVIRONMENT` | Entorno Wompi | sandbox / production |
| `NEXT_PUBLIC_APP_URL` | URL de la aplicación | https://tudominio.com |
| `NEXT_PUBLIC_APP_NAME` | Nombre de la aplicación | Krokori |

### 4.2 Generar JWT_SECRET Seguro

```bash
# Linux/Mac
openssl rand -base64 32

# Windows (PowerShell)
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) )

# Alternativa: usar una frase larga única
JWT_SECRET=mi-super-secreto-unico-para-krokori-2024
```

## 5. Proceso de Despliegue

### 5.1 Despliegue en Vercel (Recomendado)

#### Paso 1: Preparar el Proyecto

```bash
# Asegurarse de que .gitignore incluya .env.local
# Asegurarse de tener .env.example con todas las variables
```

#### Paso 2: Conectar con GitHub

1. Subir el proyecto a GitHub:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/tu-usuario/kinkfoodpizza.git
git push -u origin main
```

2. Ir a [vercel.com](https://vercel.com)
3. Click **"New Project"**
4. Importar el repositorio de GitHub
5. Configurar variables de entorno en Vercel Dashboard
6. Click **"Deploy"**

#### Paso 3: Configurar Variables en Vercel

En el dashboard de Vercel, ir a **Settings > Environment Variables**:

```env
DB_HOST=tu-servidor-mysql.com
DB_PORT=3306
DB_USER=tu_usuario
DB_PASSWORD=tu_password
DB_NAME=krokori_db
JWT_SECRET=tu-jwt-secret-seguro
JWT_EXPIRES_IN=7d
NEXT_PUBLIC_WOMPI_PUBLIC_KEY=pub_prod_xxxxx
WOMPI_PRIVATE_KEY=prv_prod_xxxxx
WOMPI_CURRENCY=COP
WOMPI_ENVIRONMENT=production
NEXT_PUBLIC_APP_URL=https://tudominio.com
NEXT_PUBLIC_APP_NAME=Krokori
```

#### Paso 4: Configurar Dominio Personalizado

1. En Vercel, ir a **Settings > Domains**
2. Agregar tu dominio (ej: `krokori.com`)
3. Configurar los DNS según las instrucciones de Vercel

### 5.2 Despliegue en Servidor Propio (VPS)

#### Requisitos del Servidor

- Ubuntu 22.04+ o similar
- Node.js 18+
- Nginx como reverse proxy
- Certbot para SSL
- MySQL 8.0+

#### Paso 1: Instalar Dependencias

```bash
# Actualizar sistema
sudo apt update && sudo apt upgrade -y

# Instalar Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Instalar PM2 para gestión de procesos
sudo npm install -g pm2

# Instalar MySQL
sudo apt install -y mysql-server

# Instalar Nginx
sudo apt install -y nginx

# Instalar Certbot
sudo apt install -y certbot python3-certbot-nginx
```

#### Paso 2: Configurar MySQL

```bash
sudo mysql_secure_installation

# Crear base de datos y usuario
sudo mysql
```

```sql
CREATE DATABASE krokori_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'krokori'@'localhost' IDENTIFIED BY 'tu_password_seguro';
GRANT ALL PRIVILEGES ON krokori_db.* TO 'krokori'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

#### Paso 3: Desplegar la Aplicación

```bash
# Clonar repositorio
git clone https://github.com/tu-usuario/kinkfoodpizza.git
cd kinkfoodpizza

# Instalar dependencias
npm install

# Importar base de datos
mysql -u krokori -p krokori_db < database/krokori_database.sql

# Configurar variables
cp .env.example .env
nano .env  # Editar con valores de producción

# Build
npm run build

# Iniciar con PM2
pm2 start npm --name "krokori" -- start

# Configurar para iniciar con el sistema
pm2 startup
pm2 save
```

#### Paso 4: Configurar Nginx

```bash
sudo nano /etc/nginx/sites-available/krokori
```

```nginx
server {
    listen 80;
    server_name tudominio.com www.tudominio.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

```bash
sudo ln -s /etc/nginx/sites-available/krokori /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### Paso 5: Configurar SSL

```bash
sudo certbot --nginx -d tudominio.com -d www.tudominio.com
```

### 5.3 Despliegue en la Nube (Railway)

## 6. Scripts de Base de Datos

### 6.1 Backup de la Base de Datos

```bash
# Crear script de backup
mysqldump -u root krokori_db > backup_krokori_$(date +%Y%m%d).sql

# Con compresión
mysqldump -u root krokori_db | gzip > backup_krokori_$(date +%Y%m%d).sql.gz
```

### 6.2 Restaurar desde Backup

```bash
# Restaurar
mysql -u root krokori_db < backup_krokori_20241120.sql

# Con gzip
gunzip < backup_krokori_20241120.sql.gz | mysql -u root krokori_db
```

### 6.3 Script Automatizado de Backup (Linux)

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/var/backups/krokori"
DB_NAME="krokori_db"
DB_USER="root"

mkdir -p $BACKUP_DIR

mysqldump -u $DB_USER $DB_NAME | gzip > $BACKUP_DIR/backup_$DATE.sql.gz

# Mantener solo los últimos 7 backups
cd $BACKUP_DIR
ls -t backup_*.sql.gz | tail -n +8 | xargs -r rm

echo "Backup completado: backup_$DATE.sql.gz"
```

### 6.4 Tareas Cron (Linux)

```bash
# Editar crontab
crontab -e

# Agregar línea para backup diario a las 3 AM
0 3 * * * /ruta/al/backup.sh >> /var/log/backup.log 2>&1
```

## 7. Troubleshooting

### 7.1 Problemas Comunes

| Problema | Solución |
|----------|----------|
| **Error de conexión a BD** | Verificar que MySQL esté corriendo y credenciales sean correctas |
| **Error 500 en API** | Revisar logs del servidor y configuración de variables |
| **Token expirado** | Recargar la página o iniciar sesión nuevamente |
| **Wompi no funciona** | Verificar llaves y que la URL de callback esté configurada |
| **Imágenes no cargan** | Verificar permisos de la carpeta `public/uploads` |

### 7.2 Verificar Logs

```bash
# Logs de desarrollo
npm run dev  # Verás los errores en consola

# Logs de PM2
pm2 logs krokori

# Logs de Nginx
sudo tail -f /var/log/nginx/error.log
```

### 7.3 Comandos de Mantenimiento

```bash
# Reiniciar la aplicación
pm2 restart krokori

# Ver estado
pm2 status

# Monitorear recursos
pm2 monit

# Limpiar caché de Next.js
rm -rf .next
npm run build
```
