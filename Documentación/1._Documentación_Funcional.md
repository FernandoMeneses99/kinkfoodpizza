# DocumentaciГіn Funcional - Krokori

## 1. DescripciГіn General del Sistema

**Krokori** es una plataforma de pedidos en lГӯnea para restaurantes tipo pizzerГӯa que permite a los clientes visualizar el menГә, realizar pedidos y pagar de forma segura. El sistema incluye un panel administrativo completo para la gestiГіn de productos, categorГӯas, Гіrdenes y configuraciГіn del negocio.

### Tipo de Sistema
- **E-commerce B2C** (Business to Consumer)
- **Plataforma de gestiГіn de pedidos** para restaurantes

### PГәblico Objetivo
- **Clientes**: Personas que desean ordenar comida en lГӯnea
- **Administradores**: DueГұos/empleados del restaurante que gestionan el negocio
- **Repartidores**: Personal de entrega (futuro)

## 2. Objetivos del Proyecto

### Objetivos Principales
1. **Digitalizar el proceso de pedidos** del restaurante
2. **Incrementar ventas** ofreciendo pedidos en lГӯnea 24/7
3. **Mejorar la experiencia del cliente** con un menГә digital intuitivo
4. **Optimizar la gestiГіn interna** del restaurante
5. **Integrar pagos en lГӯnea** de forma segura

### Objetivos EspecГӯficos
- Permitir a los clientes registrarse e iniciar sesiГіn
- Visualizar catГЎlogo de productos organizados por categorГӯas
- Gestionar una canasta de compras antes de confirmar el pedido
- Procesar pagos mediante pasarela de pagos (Wompi)
- Notificar el estado del pedido en tiempo real
- Gestionar Гіrdenes desde el panel administrativo
- Generar reportes de ventas y estadГӯsticas

## 7. Reglas de Negocio

### 7.1 GestiГіn de Usuarios
- **RN-001**: El email debe ser Гәnico en el sistema
- **RN-002**: La contraseГұa debe tener mГӯnimo 6 caracteres
- **RN-003**: Solo existe un rol de administrador en el sistema
- **RN-004**: Los clientes pueden tener mГәltiples direcciones

### 7.2 GestiГіn de Productos
- **RN-005**: El precio debe ser mayor a 0
- **RN-006**: El precio de oferta debe ser menor al precio original
- **RN-007**: Un producto debe pertenecer a una categorГӯa
- **RN-008**: Los productos inactivos no se muestran en el menГә pГәblico

### 7.3 GestiГіn de Pedidos
- **RN-009**: El subtotal se calcula automГЎticamente (suma de productos)
- **RN-010**: El costo de domicilio se calcula segГәn reglas configuradas
- **RN-011**: El total = subtotal + costo_domicilio - descuento
- **RN-012**: Solo se pueden crear Гіrdenes con productos disponibles
- **RN-013**: El nГәmero de orden es Гәnico y se genera automГЎticamente

### 7.4 Proceso de Pago
- **RN-014**: Para pago en lГӯnea, la orden queda pendiente hasta confirmar el pago
- **RN-015**: Si el pago falla, la orden se mantiene en estado pendiente
- **RN-016**: Para pago contra entrega, la orden se confirma directamente

### 7.5 Descuentos y Promociones
- **RN-017**: Los descuentos pueden ser montos fijos o porcentajes
- **RN-018**: Los descuentos no pueden superar el subtotal

### 7.6 Horarios de AtenciГіn
- **RN-019**: El sistema puede configurarse con horarios de atenciГіn
- **RN-020**: Las Гіrdenes fuera de horario se pueden programar para el siguiente dГӯa
