### 3.2 Convenciones de Código

#### Nomenclatura de Archivos
- **Componentes**: `PascalCase.tsx` (ej: `Navbar.tsx`)
- **Hooks**: `camelCase` con prefijo `use` (ej: `useAuth.ts`)
- **Utilidades**: `camelCase.ts` (ej: `db.ts`)
- **API Routes**: `route.ts`

#### Nomenclatura de Variables
- **Funciones**: `camelCase` (ej: `createUser`, `getProducts`)
- **Constantes**: `UPPER_SNAKE_CASE` (ej: `JWT_SECRET`)
- **Clases**: `PascalCase` (ej: `UserService`)

#### Tipado TypeScript
- **Interfaces**: Prefijo `I` (ej: `IUser`, `IProduct`)
- **Types**: Sin prefijo (ej: `type OrderStatus`)
- **Props**: Prefijo `Props` (ej: `ProductCardProps`)

### 4.2 Descripción de Tablas

#### `usuarios`
```sql
CREATE TABLE usuarios (
  id_usuario INT PRIMARY KEY AUTO_INCREMENT,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  nombre VARCHAR(100) NOT NULL,
  telefono VARCHAR(20),
  rol ENUM('admin', 'cliente', 'repartidor') DEFAULT 'cliente',
  activo TINYINT DEFAULT 1,
  email_verificado TINYINT DEFAULT 0,
  ultimo_login DATETIME,
  fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_email (email),
  INDEX idx_rol (rol)
);
```

#### `categorias`
```sql
CREATE TABLE categorias (
  id_categoria INT PRIMARY KEY AUTO_INCREMENT,
  nombre VARCHAR(100) NOT NULL,
  slug VARCHAR(100) UNIQUE NOT NULL,
  descripcion TEXT,
  icono VARCHAR(50),
  orden INT DEFAULT 0,
  activo TINYINT DEFAULT 1,
  INDEX idx_slug (slug),
  INDEX idx_orden (orden)
);
```

#### `productos`
```sql
CREATE TABLE productos (
  id_producto INT PRIMARY KEY AUTO_INCREMENT,
  categoria_id INT NOT NULL,
  nombre VARCHAR(150) NOT NULL,
  slug VARCHAR(150) UNIQUE NOT NULL,
  descripcion TEXT,
  precio DECIMAL(10,2) NOT NULL,
  precio_oferta DECIMAL(10,2),
  imagen VARCHAR(255),
  ingredientes TEXT,
  disponible TINYINT DEFAULT 1,
  destacado TINYINT DEFAULT 0,
  tiempo_preparacion INT DEFAULT 30,
  stock INT DEFAULT 0,
  activo TINYINT DEFAULT 1,
  FOREIGN KEY (categoria_id) REFERENCES categorias(id_categoria),
  INDEX idx_slug (slug),
  INDEX idx_categoria (categoria_id),
  INDEX idx_disponible (disponible)
);
```

#### `ordenes`
```sql
CREATE TABLE ordenes (
  id_orden INT PRIMARY KEY AUTO_INCREMENT,
  usuario_id INT,
  repartidor_id INT,
  numero_orden VARCHAR(20) UNIQUE NOT NULL,
  estado ENUM('pendiente','pagado','confirmado','preparando','listo','en_camino','entregado','cancelado') DEFAULT 'pendiente',
  estado_pago ENUM('pendiente','pagado','fallido','reembolsado') DEFAULT 'pendiente',
  nombre_cliente VARCHAR(100) NOT NULL,
  telefono_cliente VARCHAR(20) NOT NULL,
  email_cliente VARCHAR(255),
  direccion_entrega TEXT NOT NULL,
  barrio VARCHAR(100),
  referencia_direccion TEXT,
  subtotal DECIMAL(10,2) DEFAULT 0,
  costo_domicilio DECIMAL(10,2) DEFAULT 0,
  descuento DECIMAL(10,2) DEFAULT 0,
  total DECIMAL(10,2) DEFAULT 0,
  metodo_pago ENUM('wompi','nequi','daviplata','contra_entrega') DEFAULT 'contra_entrega',
  observaciones TEXT,
  FOREIGN KEY (usuario_id) REFERENCES usuarios(id_usuario),
  INDEX idx_numero_orden (numero_orden),
  INDEX idx_estado (estado),
  INDEX idx_usuario (usuario_id)
);
```

## 5. API Endpoints

### 5.1 Autenticación `/api/auth/`

| Método | Endpoint | Auth | Descripción |
|--------|----------|------|-------------|
| POST | `/api/auth/register` | No | Registrar nuevo usuario |
| POST | `/api/auth/login` | No | Iniciar sesión |
| POST | `/api/auth/logout` | Sí | Cerrar sesión |
| GET | `/api/auth/me` | Sí | Obtener usuario actual |
| POST | `/api/auth/refresh` | Refresh Token | Refrescar access token |

#### POST /api/auth/register
**Request:**
```json
{
  "email": "usuario@ejemplo.com",
  "password": "contraseña123",
  "nombre": "Juan Pérez",
  "telefono": "3001234567"
}
```
**Response (201):**
```json
{
  "success": true,
  "message": "Usuario registrado exitosamente",
  "user": {
    "id": 2,
    "email": "usuario@ejemplo.com",
    "nombre": "Juan Pérez",
    "rol": "cliente"
  }
}
```

#### POST /api/auth/login
**Request:**
```json
{
  "email": "usuario@ejemplo.com",
  "password": "contraseña123"
}
```
**Response (200):**

│  ┌─────────┐    POST /api/auth/login    ┌─────────────────┐ │
│  │ Cliente │ ─────────────────────────▶ │ Generate JWT    │ │
│  │         │ ◀───────────────────────── │ Generate UUID   │ │
│  └─────────┘    accessToken + refreshToken │ (saved in DB) │ │
│                 └─────────────────┘                       │
│                                                              │
│  2. ACCESS TOKEN (7 días)                                    │
│  ┌─────────┐    GET /api/resource     ┌─────────────────┐ │
│  │ Cliente │ ─── Authorization: Bearer ─▶ Validate JWT   │ │
│  │         │ ◀── Response + data       │ Extract user    │ │
│  └─────────┘                            └─────────────────┘ │
│                                                              │
│  3. REFRESH (cada 7 días)                                    │
│  ┌─────────┐    POST /api/auth/refresh ┌─────────────────┐ │
│  │ Cliente │ ─────────────────────────▶ │ Verify refresh  │ │
│  │         │ ◀───────────────────────── │ Generate new    │ │
│  └─────────┘    new accessToken +        │ accessToken     │ │
│                 new refreshToken          └─────────────────┘ │
│                                                              │
│  4. LOGOUT                                                    │
│  ┌─────────┐    POST /api/auth/logout  ┌─────────────────┐ │
│  │ Cliente │ ─────────────────────────▶ │ Invalidate     │ │
│  │         │ ◀───────────────────────── │ all refresh     │ │
│  └─────────┘    success                 │ tokens for user │ │

└─────────────────────────────────────────────────────────────┘

## 9. Procedimientos Almacenados

### 9.1 sp_dashboard_stats
```sql
DELIMITER //
CREATE PROCEDURE sp_dashboard_stats()
BEGIN
  SELECT 
    (SELECT COUNT(*) FROM ordenes WHERE DATE(fecha_registro) = CURDATE()) as ordenes_hoy,
    (SELECT COUNT(*) FROM ordenes WHERE estado = 'pendiente') as ordenes_pendientes,
    (SELECT COUNT(*) FROM productos) as total_productos,
    (SELECT COUNT(*) FROM usuarios WHERE rol = 'cliente') as total_clientes,
    (SELECT COALESCE(SUM(total), 0) FROM ordenes WHERE DATE(fecha_registro) = CURDATE() AND estado_pago = 'pagado') as ventas_hoy;
END //
DELIMITER ;
```

### 9.2 sp_productos_mas_vendidos
```sql
DELIMITER //
CREATE PROCEDURE sp_productos_mas_vendidos(IN top_n INT)
BEGIN
  SELECT 
    p.nombre,
    p.imagen,
    SUM(od.cantidad) as total_vendido,
    SUM(od.subtotal) as ingresos
  FROM orden_detalles od
  JOIN productos p ON od.producto_id = p.id_producto
  JOIN ordenes o ON od.orden_id = o.id_orden
  WHERE o.estado_pago = 'pagado'
  GROUP BY p.id_producto
  ORDER BY total_vendido DESC
  LIMIT top_n;
END //
DELIMITER ;
```
